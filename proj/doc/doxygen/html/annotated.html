Data Structures
Here are the data structures with brief descriptions:
 CAnimSpriteAn AnimSprite is composed of a sprite which points to the current frame bitmap. The map is switched when it's time to show the next animation (according to the frames_per_map attribute). According to the direction of the animated sprite, it may be flipped horizontally
 CButtonA Button is composed by two sprites - one with solid color, and one with colored frame to represent the hover - and a text that will be written in its center (with the desired font)
 CFontA Font is composed of each letter sprite, in its original size and zoomed 2x
 CGameA Game holds information about the mouse, the player, the map and the font to write the buttons and timer. Clues and endgame messages are stored in a panel that is mutable during the game
 CInstructionsInstructions is a panel wrapper with big padding between the text, aligned left, and a corresponding image to each text string
 CMapA Map holds information about the blocks for each position, and the sprites corresponding to each one, in original size and zoomed with the corresponding zoom factor. Since the zoomed map moves, the relative position of the map top left is also stored
 CMenuThe Menu holds buttons to the different game states, and a title. Stores the instructions and the settings but not the game
 CPanelA Panel is composed of a title, a solid background, text strings and a close button
 CPlayerA Player is composed of its sprite and a series of attributes related to the its movement in the map, e.g x_abs and y_abs representing the absolute position in the zoomed map, and speed the step of the sprite per movement
 CResourcesResources is a wrapper to the main objects allocated and manipulated by the app state handlers
 CSettingsSettings is a panel wrapper with aditional buttons (options) at the bottom to the first panel string
 CSpriteA Sprite holds the width, height, x and y positions and the bitmap of an object that shall be drawn on the screen. If x are y, negative mean that the top left of the sprite is indeed ouside of the screen, and only a part of the sprite is shown
